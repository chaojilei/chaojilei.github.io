产品简析-逗拍

  • 产品分析

那天老妈分享给我一段视频。亲戚的大头贴充当动态头像，在视频里边蹦蹦跳跳，非让我也给她装这个软件。于是乎搜索了一下，发现是个叫“逗拍”的应用。而且这个应用这两天在AppStore里边排位还非常高，于是在简单体验了一番（iOS），做个简单分析：

  • 产品简介：
    逗拍 -大头视频制作神器。2014年12月2日上线，目前发布了4个版本。主要功能是大头视频制作与分享。类型应该类似于以往火爆过的脸萌、魔漫相机等。

  • 目标用户：
    喜欢新奇、热爱分享的用户。

  • 使用场景：

    • 用户点开应用，选择视频模板，选择大头照片，生成大头视频，分享大头视频
    • 用户点开应用，播放本机生成过的视频给周围朋友观看。

  • 功能与交互概况：

  • App更新记录：
    App2014-12-2上线，此后经历了4个版本。

    • 版本 1.0.3 (2015年1月29日)
      修复了 1.0.2 视频下载失败以及 QQ 分享失败的 Bug
      强烈推荐安装了 1.0.2 版本的用户马上更新
    • 版本 1.0.2 (2015年1月27日)
      新增保存视频到手机相册功能，你可以很方便分享给好友；
      允许同时下载2个视频，好用不费电！
      修复些许bug，工程师累晕在 iMac 旁；
    • 版本 1.0.1 (2014年12月21日)
      分享更简单方便
      新增各种逗趣模板
      增加了评论功能
      修复某些你肉眼看不到的bug
      支持 iOS6
    • 版本 1.0 (2014年12月2日)

  • AppStore下载数据：
    从图中可以看出进入一月后，下载量明显上升，在特别是在摄影类别中名列前茅。可以说产品进入了用户飞速增长期。

  • 产品亮点：
    • 重视用户反馈
      用户反馈的入口非常明显，说明此产品非常重视用户的反馈（至少给用户这样的感觉），这对于新生产品非常重要。
    • 上线时机很好
      随着智能手机的普及，以及网络条件改善，短视频类的分享应用十分火爆。逗拍这种diy性质的产品，前有魔漫相机、脸萌在图片端的成功，同时结合最近即将到来的情人节、春节，做好运营，新增用户量应该还会有大量的增长。
  • 问题与瓶颈：
    • 产品大方向：
      • 竞争壁垒较低
        目前短视频类的产品很多，例如美拍、微视等。如何面对竞品迭代可能的模仿，是这款新生App首先要面对的。
      • 如果留住用户
        就像前面说的，单纯的工具类应用前有魔漫相机、脸萌，都曾昙花一现。基于模板的应用，如何产生足够新鲜的UGC，如何能够留住用户才是这款产品发展的关键。从逗拍在“我的视频”中的产品页面布局肯已看出其以后也可能要引入社交功能，让用户在App本身分享，评论、点赞，从而增强用户黏性，具体效果只能持续关注。
    • 产品本身：
      • 产品稳定性不足
        虽然是新产品，但是从下载数据可以看到新增用户非常多，初期的版本稳定性十分重要。在我的使用体验过程中，多次出现闪退的情况，体验需要加强。同时在1.27,1.29连发了2个版本，说明之前的版本也是带有明显漏洞的。
      • 模板的选取
        这款应用的特色在于模板与大头贴。目前的模板数量一般，主要以搞笑风格为主。而且每次点击需要在线下载模版，而无法提前预览，这点比较伤用户。后期应该压缩模板到App本身，同时加入更多精美的模板，部分优质内容可以考虑收费。
      • 大头贴的制作
        在体验过程中，制作出合适的大头贴并不容易。目前的流程是选择脸型（部分模板只有一个）-》选择拍照或相册-》调整脸型（大小、方向）。现有的脸型模板太少了，很多照片套进去效果都不够好，需要调整很久。这里需要加强人脸识别算法的优化，自动识别脸型，根据实际的照片推荐或生成最合适的脸型模板，这样的流程才能达到最好的用户体验。
